01 Product Context
Kun Troll — Homeware Hub is a universal home remote controller for iPhone. It replaces every physical remote in the house with a single, character-driven interface. The benchmark is Roomie Remote X (400K+ devices, the category leader). Kun Troll matches its depth and exceeds it in three areas: AI-assisted device discovery (Troll Scout), onboarding personality, and visual identity.

The Three Pillars

AI-First

Troll Scout automatically searches the web for device commands when a device has no library match. Users never hit a dead end.

Character-Driven

Kun Troll is present throughout — in onboarding, success states, errors, idle screens. The app has a personality. Not a mascot.

Visually Distinctive

Dark violet and lime. Rounded shapes. Space Mono for data. A look that says premium without saying corporate.

02 Character Specification — Kun Troll
⚠ This specification is locked. Every element below is final. Do not improvise, simplify, or add features not listed. Kun Troll must look identical across every screen, widget, and onboarding step.

Anatomy — Build Exactly This

PartExact Specification
Head shape Wide oval — wider than it is tall. Horizontal stretch roughly 1.4× the vertical. Low-slung, grounded proportions. NOT a circle, NOT a tall oval. Think "friendly pebble."
Hair spikes Exactly 3 upward-pointing spikes on top of the head. Medium thickness — not thin needles, not thick blobs. Slight forward lean. Troll signature. Do not add more or fewer.
Ears Small pointed troll ears, one on each side of the head. Positioned low — roughly at the midpoint of the head height. Slightly angled outward. Do not enlarge them.
Eyes Two large circular eyes that sit close together (not wide apart). Each eye: white sclera → #C8FF00 lime-green iris (large, fills most of the eye) → dark circular pupil centred → single small white highlight dot at top-right of the pupil. The iris should feel bright and alive.
Mouth Gentle upward curved smile. No teeth visible. Width approximately 40% of the head width. Default state is always a smile. Only Sleepy gets a slight downturn.
Eyebrows NONE. Do not add eyebrows. Do not add brow ridges. Zero.
Nose NONE. Do not add a nose, nostrils, or nose line. Zero.
Body Head only. No neck, no torso, no arms, no legs. The character is a floating head with hair. This is intentional and iconic.
Fill colour Skin: #6B28EE violet. Eyes: #C8FF00 lime. No gradients on the character skin. Flat fill with outline.
Outline 1.5–2px dark #1A1A2E outline on all shapes. Clean and consistent weight.

The 5 Expressions — Use Only These

No other expressions exist. If a new product state requires an expression, define it here first. Do not improvise.

ExpressionExact Visual ChangesWhen to Use
Happy Eyes fully open and bright. Full curved smile. Character floats gently. This is the neutral/default state — every screen that doesn't specify an expression uses Happy. App open, home dashboard, device online, command sent, general navigation
Wow Eyes enlarged to ~130% of normal size. Mouth open in a rounded "O" shape (not a smile, not a frown — pure surprise). Character leans slightly back. Trigger a scale pulse: 1.0 → 1.15 → 1.0 in 200ms. Device discovered, Troll Scout finds commands, scene fires successfully with all devices responding
Wink Right eye is closed (simple curved line). Left eye remains fully open. Head tilts a few degrees to the right. Playful and knowing. Blink animation does NOT apply — the eye stays closed. Troll Scout actively searching, scene saved/confirmed, onboarding step completed
Sleepy Both eyelids drooped to 60% closed (upper lid covers top 60% of the eye). Eyes are still visible but heavy. Mouth slightly downturned — not a frown, just neutral. Overall droop effect. Transition in slowly: 1s ease-in. Device offline, command failed, idle 5+ minutes, not found states
Love Eyes replaced with sparkle/star shapes (4-point star or heart outline — not filled circles). Huge smile, slightly wider than default. Small floating hearts above the head (2–3 hearts, different sizes). Scale pulse on entrance. Onboarding complete, first ever scene fired, user returns after 7+ days away

Animation Rules

AnimationParametersWhere It Applies

BlinkLids close in 80ms, open in 120ms. Trigger randomly every 5–7s.Happy and Wink only. Not Wow (eyes too wide). Not Sleepy (already drooped).

Placement Rules

  • Hero zone: Top 30% of screen, horizontally centred. Full-size (120–160pt diameter). Used on: Home Dashboard, Onboarding all steps, Troll Scout screens.
  • Corner badge: 40×40pt circle version, anchored bottom-right of the screen content area (above tab bar). Used on all remote view screens (TV, Receiver, Lighting, Apple TV). Winks on command success.
  • Empty state: Centred, Sleepy expression, helper text directly below. Never leave an empty state with just text and no character.
  • Loading state: Wink expression, subtle hat spike rotation animation, "Searching..." label below in Space Mono.
  • Character always renders on top of all other UI elements. Never clip or place behind cards.
  • Minimum size: 48pt diameter. At this size only eyes and smile are legible — this is acceptable.
  • Glow effect: radial gradient rgba(107,40,238,0.3) → transparent, radius 80pt, centred on the character. Applied only in hero zone. Not on corner badge. Not anywhere else.

04 Layout Rules
  • Target device: iPhone 15 Pro — 390×844pt. Design to this canvas.
  • Hero zone: Top 30% (roughly 0–253pt from top). Kun Troll + key headline live here.
  • Interaction zone: Bottom 60% of screen. All tappable primary actions must be thumb-reachable here.
  • Safe areas: Top 59pt (Dynamic Island), bottom 34pt. Nothing critical behind safe areas.
  • Screen edge margins: 20–24pt left and right. Consistent.
  • Numeric data: Always Space Mono. Volume levels, IP addresses, port numbers, command codes.
  • Cards: Full-width minus edge margins. 16pt internal padding. 20pt corner radius. Card border rgba(180,160,255,0.18).
  • No gradients on backgrounds. The background is flat #1A1A2E. Only the character glow uses a gradient.
  • Swipe left/right on Remote tab to navigate between devices in the same room.
  • Long press on Room card reveals a quick scene launcher overlay.
  • Long press on a command button reveals command details (name, source, confidence).
05 Troll Scout — AI Feature (Phase 2)
Troll Scout is the AI layer that searches the web for device control commands when a device has no match in the local library. It is the biggest differentiator vs. Roomie Remote X. No mainstream remote app does this.

What It Does

  • Triggered automatically when a discovered device has no library match.
  • Claude API (claude-sonnet-4-20250514) + web_search searches manufacturer docs, IRDB.tk, RemoteCentral, GitHub, Home Assistant.
  • Extracts and normalises commands into the app's schema. Assigns confidence score per source.
  • Presents results for user review — commands never fire automatically.
🔒 CRITICAL GUARDRAIL: AI-fetched commands are NEVER executed automatically. NEVER saved without explicit user confirmation. The confirmation screen is mandatory and cannot be bypassed. This must be visually clear in every Troll Scout screen.

Command Source Badges (Space Mono · Pill)

Badge TextColourMeaning
manual#4ADE80 greenUser entered this command themselves
library#C8FF00 limeFrom the built-in device library
ai_fetched#A78BFA violetFound by Troll Scout — requires confirmation

Confidence Dots

Each command row shows 3 dots filled based on confidence score. 3 dots = official docs (0.9+). 2 dots = community source (0.6–0.89). 1 dot = forum/unverified (below 0.6). Dots use lime when filled, muted when empty.

Troll Scout Expression Flow

  • Searching: Wink expression. Spikes rotate subtly. Text cycles source names: "Searching IRDB... Checking manufacturer docs..."
  • Found: Wow expression + pulse. "Found 47 commands for Yamaha RX-A870"
  • User confirms all: Happy expression. "Device is ready!"
  • Not found: Sleepy expression. "Troll Scout couldn't find commands for this device." Manual entry CTA below.

07 What Every Screen Must Have
  • Kun Troll in the correct expression and correct placement (hero zone OR corner badge). No exceptions.
  • Bottom tab bar on all screens except Onboarding and modal flows (Add Device, Troll Scout).
  • Correct active tab highlighted with lime accent and underline dot.
  • All numeric values (volume, IP, port, time, confidence) in Space Mono.
  • Real device names from the confirmed device list — not generic "Device 1".
  • Real scene names — Cinema, Good Night, Good Morning, Music, All Off.
  • Card borders: rgba(180,160,255,0.18). Never a solid line.
  • Character glow (radial violet gradient) in hero zone only. Not on corner badge or empty states.
  • Haptic feedback implied for all command buttons (note in code comments).
  • iPhone 15 Pro safe areas respected — Dynamic Island top 59pt, home indicator 34pt.

What to Never Do

  • Never add eyebrows or a nose to Kun Troll. Never add a body.
  • Never use light mode or white backgrounds on any screen.
  • Never use San Francisco font. Always Nunito + Space Mono.
  • Never fire an AI-fetched command without showing the confirmation screen first.
  • Never use pure black (#000000) for any background. Always #1A1A2E.
  • Never place the character behind other UI elements. Always in front.
  • Never use the violet glow outside the character hero zone.
  • Never use more than 5 expressions — if you need another, that's a product decision first.
  • Never use generic placeholder names: no "Living Room 1", no "Scene A".
  • Never use 6 spikes on Kun Troll's head. Exactly 3.
